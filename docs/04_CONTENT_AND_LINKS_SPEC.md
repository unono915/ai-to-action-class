## 7. 교과별 사례 기본 데이터

### 체육

- 모델: 자세
- 대상: 스쿼트·준비 자세·팔 위치
- 레이블: 바른 자세 / 확인 필요
- 행동: 교정 문구·시범 이미지
- 주의: 체형과 촬영 각도 차이

### 국어

- 모델: 음성
- 대상: 발음·음운 변동·낭독
- 레이블: 발음 A / 발음 B / 재확인
- 행동: 관련 규칙·다시 읽을 문장
- 주의: 정답 판정기가 아닌 탐구 활동

### 음악

- 모델: 음성
- 대상: 악기 소리·리듬·강약
- 레이블: 악기 A / B / C
- 행동: 악기 정보·다음 연주 안내
- 주의: 주변 소음과 혼합음

### 과학

- 모델: 이미지
- 대상: 암석·잎 상태·색 변화
- 레이블: 종류 또는 상태
- 행동: 관찰 질문·추가 실험
- 주의: 사진만으로 확정하기 어려운 성질

### 미술

- 모델: 이미지
- 대상: 색상 계열·구도·표현 기법
- 레이블: 유형별
- 행동: 관련 작품·표현 활동
- 주의: 작품 가치 평가 금지

### 기술·가정

- 모델: 이미지 또는 자세
- 대상: 재료·공구·분리수거·안전 행동
- 레이블: 종류·안전 여부
- 행동: 사용법·안전 수칙
- 주의: 위험한 장치 작동을 모델에만 의존하지 않음

### 진로·창체

- 모델: 이미지
- 대상: 직업 도구·작업 환경·안전 행동
- 행동: 직업 정보·캠페인
- 주의: 사람의 적성·성격 판단 금지

## 8. 프롬프트 빌더 기본 템플릿

```text
너는 학생의 [교과 개념] 이해를 점검하는 AI 학습 도우미이다.

대상은 [학년] 학생이다. 말투는 [말투]로 유지한다.

학생에게 정답을 바로 알려주지 말고, 먼저 학생이 자신의 말로 개념을 설명하게 한다.

학생의 답변에서 다음 내용을 확인한다.

1. [핵심 요소]
2. 빠진 조건이나 설명
3. 다음과 같은 자주 나타나는 오개념: [오개념]
4. 적절한 예시
5. 새로운 상황에 적용할 수 있는지

답변이 부족하면 정답을 바로 말하지 말고, 다시 생각할 수 있는 질문이나 짧은 힌트를 제공한다.

첫 질문은 다음과 같이 시작한다.
“[첫 질문]”

대화의 마지막에는 학생이 [마지막 산출물]을 자신의 말로 작성하게 한다.
```

빈 필드는 자연스러운 기본 문장으로 처리하되, 내용을 임의로 발명하지 않는다.

## 9. 자료 기반 Gem 지침

```text
답변은 연결된 자료를 우선 근거로 작성한다.

자료에서 근거를 확인할 수 없는 내용은 추측하지 말고
“제공된 자료에서는 확인할 수 없습니다.”라고 답한다.

가능한 경우 답변의 근거가 된 자료나 항목을 함께 제시한다.
```

주의 문구:

> 자료를 연결하면 자료에 근거한 답변을 유도하고 오류 가능성을 줄이는 데 도움이 되지만, AI의 오류가 완전히 사라지는 것은 아닙니다.

## 10. 자산 규칙

- 최종 메인 이미지는 `public/assets/hero/`에 저장
- 화면 캡처는 `public/assets/screenshots/`
- QR 이미지는 가능하면 런타임 생성보다 정적 또는 구성 데이터 기반
- 자산이 없으면 깨진 이미지 대신 의미 있는 placeholder
- 저작권이 불분명한 외부 이미지를 임의로 포함하지 않음


## 최종 메인 이미지

- 파일 경로: `public/assets/hero/main-hero.png`
- 이미지가 없으면 CSS placeholder 사용
- 웹페이지 제목과 행사 정보는 이미지 위 또는 아래 HTML 텍스트로 표시
- 이미지 자체에는 글자·로고를 넣지 않음
