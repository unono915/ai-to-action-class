## 2. 대화에서 확정된 핵심 설계 의도 (문서 00~06과 다르거나 추가된 것)

1. **발표 모드 = 슬라이드 덱.** 발표자 메모·별도 기능을 넣는 화면이 아니라,
   실습 모드와 같은 내용을 발표용 슬라이드처럼 크게 띄우는 용도.
   타이머·전체화면·권장시간 배지만 유지. `presentationSlides.ts`가 소스.
2. **2단계는 '수업 나눔'.** '수업 철학'이라는 프레임을 앞세우지 않는다.
   실제 수업 「이미지 분류 모델로 프로그램 움직이기」(2026-06-05 고1 연구수업)를
   **시간순으로 따라가는 사례 나눔이 뼈대**이고, 철학은 생각이 실제 담긴 장면에만
   `insight`(「이 장면에 담긴 생각」) 한 줄로 스민다. 철학을 위해 사례를
   끼워 맞추지 않는다. 소스: `lessonNarrative.ts` (18개 장면).
3. **행사 진행 구조: PPT → 웹 → PPT.** 발표자 PPT(8장)로 열고, PPT 5번
   슬라이드의 링크로 참가자가 이 사이트에 접속, 모든 나눔·실습은 웹에서,
   핵심 마무리는 웹 STEP8에서 전달한 뒤 Q&A는 다시 PPT에서 진행한다.
   그래서 **사이트 안에 참가자용 QR은 넣지 않는다**.
4. **5단계 아이디어 코치 Gem 섹션은 버튼 2개만.** 실제 Gem은 긴 질문 순서로
   진행되지 않으므로 정리 입력칸·저장 기능 없이 "간단한 질문으로 아이디어
   얻기" 안내 + Gem 열기 + Padlet 공유만.
5. **타이머는 스톱워치 유지** (카운트다운 불필요). 발표 모드 하단에
   단계별 `권장 N분` 배지 표시.
6. 실습은 각자 노트북 진행이라 **인원수(40명)와 소요 시간은 무관** —
   운영 포인트는 순회 코칭과 예시 모델 안전망.
7. **3·4단계 실습 페이지는 행동 우선 구조.** 3단계의 산출물은 O·X·배경 모델과
   공유 URL이며, 상단의 Teachable Machine 버튼에서 시작해 4개 작업 카드
   (프로젝트 → 데이터 → 학습·시험 → 내보내기)를 따라간다. 4단계의 산출물은
   손동작에 따라 문구·이미지가 실제로 바뀌는 장면이며, 상단의 행동 연결 버튼에서
   시작해 3개 작업 카드(URL 연결 → 행동 설정 → 결과 확인)를 따른다. 화면 캡처는
   별도 세로 열이 아니라 해당 작업 카드 안에 배치한다. 우측 완료 패널, 오류 기록란,
   URL 복사 단독 체크, 별도 예시 표는 하단의 공통 `이 단계 완료`와 중복되므로
   사용하지 않는다. 예시 모델은 막혔을 때만 쓰는 보조 안전망으로 표시한다.
8. **6단계 Gem 만들기는 메타 프롬프트 다운로드 방식.** 짧은 Gem 지침을
   브라우저에서 바로 만드는 방식은 사용하지 않는다. 교사가 입력한 교과·학년·개념·
   핵심 요소·오개념 등을 정교한 메타 프롬프트에 채워 `.md`로 내려받고, 원하는
   생성형 AI에 입력하여 ① Gem 요청사항 ② Gem 지식 자료 초안 ③ 교사 검토 메모를
   만든다. 사이트는 AI API를 호출하거나 입력값을 전송하지 않는다. 생성된 지식은
   교과서·수업 자료와 대조한 뒤 사용하며, 학습 완료 코드는 보안·평가 수단이 아닌
   보조 확인 수단으로만 안내한다. 템플릿 소스: `gemMetaPromptTemplate.md`.
9. **7단계는 근거 자료 연결을 다루는 선택 확장.** 6단계에서 만든 Gem만으로
   실습을 마쳐도 되며, 근거 자료가 특히 중요하거나 여러 자료를 계속 관리할 때
   선택한다. 자료가 적으면 검토한 파일을 Gem 지식에 바로 추가하고, 확장하려면
   수업 자료·공식 출처를 **Gemini Notebook(구 NotebookLM)**에 모아 교사가
   원문을 검토한 뒤 노트북 자체를 Gem 지식에 연결한다. Notebook 안의 Deep
   Research는 보고서와 출처 후보를 모으는 선택 수단이지 검증을 대신하지 않는다.
   목적은 답변의 근거 범위를 좁혀 할루시네이션 가능성을 줄이는 것이며,
   무오류를 보장하지 않는다는 한계와 자료 밖 응답 제한을 함께 안내한다.
   소스: `groundedGem.ts`.
10. **8단계는 실천 계획 입력이 아니라 연수 핵심 메시지 정리.** 하버드의 맞춤형
   AI 튜터 연구와 MIT Media Lab의 범용 LLM 글쓰기 프리프린트를 나란히 보되,
   과제·대상·설계가 달라 직접 비교할 수 없다는 한계를 밝힌다. 결론은 연구 간
   단순 우열이 아니라, AI를 학습 도우미로 쓸지 작업을 대신하는 도구로 쓸지
   결정하는 **교사의 역할 설계**가 중요하다는 메시지다. 실습 모드와 발표 모드는
   `closing.ts`의 같은 핵심 문구를 사용한다.
11. **1단계는 연수의 목적과 여정을 한 화면에 보여 주는 시작점.** 이미지가 먼저
   화면을 점유하던 구조를 버리고, 왼쪽에 “AI의 판단을 학생의 사고와 수업 행동으로
   연결하는 흐름을 교사가 설계한다”는 메시지·행사 정보·시작 버튼을 두고 오른쪽에
   교사 중심의 새 히어로 이미지를 배치한다. 아래에는 네 가지 여정(판단 만들기 →
   행동 연결하기 → 교과 확장하기 → 학습 Gem 설계하기)과 준비물을 2열로 정리한다.
   `수업 나눔부터 시작`은 2단계로 이동하고, `발표 모드 열기`는 1단계 표지부터 연다.
   실습 화면과 발표 모드는 `startPage.ts`의 같은 문구와 이미지를 사용한다.

## 3. 데이터 구조 (2차 개발 이후 추가분)

```text
src/data/
  lessonNarrative.ts   2단계 수업 나눔 18장면 (story/insight/quote/image/photo)
                       + lessonOverview(지도안 개요) + lessonTools(도구 4종 효과)
  lessonImages.ts      교안 PDF에서 추출한 슬라이드 13장 (public/assets/lesson/)
  classPhotos.ts       실제 수업 현장 사진 6장 (public/assets/class/, EXIF 제거됨)
  presentationSlides.ts 발표 모드 전체 슬라이드 (2단계는 lessonNarrative 매핑)
  screenshots.ts       실습 안내용 도구 캡처 12장 (public/assets/screenshots/)
  practiceWorkflows.ts 3·4단계 작업 카드·결과 목표·5단계 확장 행동
  groundedGem.ts       7단계 선택 경로·Notebook 확장 흐름·지침·검토 항목
  closing.ts           8단계 연구 요약·설계 요소·최종 메시지
  startPage.ts         1단계 핵심 메시지·히어로 자산·네 가지 여정·준비물
```

- 이미지 표시 규칙: 장면에 `photo`(현장 사진)가 있으면 `image`(교안)보다 우선.
- 교안 이미지 순서는 원본 PDF 페이지 순서(=수업 진행 순서)와 일치시켜 두었음.
