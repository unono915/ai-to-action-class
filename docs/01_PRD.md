## 2. 핵심 운영 결정

### 하나의 주소에서 모드 전환

발표자와 참가자가 같은 주소를 사용한다.

- 기본 주소: `https://unono915.github.io/ai-to-action-class/`
- 참가자 모드: `?mode=practice`
- 발표자 모드: `?mode=present`

권장 예시:

```text
https://unono915.github.io/ai-to-action-class/?mode=practice
https://unono915.github.io/ai-to-action-class/?mode=present
```

### 이 방식의 장점

1. 참가자에게 하나의 QR 코드만 제공할 수 있다.
2. 발표 내용과 실습 안내가 같은 콘텐츠를 공유한다.
3. `/present` 별도 경로보다 GitHub Pages의 SPA 새로고침 문제를 줄일 수 있다.
4. 발표 모드와 실습 모드가 같은 단계 번호를 공유할 수 있다.
5. 이후 수정할 때 한 곳만 관리하면 된다.

### 모드 전환 규칙

- 기본 진입은 `practice`
- 상단 우측에 `발표 모드 / 실습 모드` 전환 스위치 제공
- 모드 전환 시 URL query parameter 변경
- 새로고침 후에도 현재 모드 유지
- URL 값이 localStorage 값보다 우선
- 발표자 타이머·발표자 메모는 발표 모드에서만 노출
- 실습 완료 체크와 상세 도움말은 실습 모드에서 강조

## 6. 페이지별 핵심 요구사항

### 시작하기

- 메인 이미지
- 세션 제목과 한 줄 설명
- 오늘 가져갈 결과물 4개
- 준비물 체크
- 전체 흐름
- 발표 모드 시작
- 실습 모드 시작

### 왜 이런 수업인가

발표 장면:

1. 학생은 왜 배우냐고 묻는다.
2. 지식은 삶과 문제 해결로 확장되어야 한다.
3. AI 시대에는 기능·원리·활용 판단이 필요하다.
4. 분류 모델 판단에서 수업이 끝날 수 있다.
5. 판단을 행동으로 연결하면 문제 해결 구조가 된다.
6. 모델 훈련과 행동 설정은 별개의 일이다.
7. AI가 인간보다 항상 잘 판단하는 것은 아니다.
8. 학생의 질문이 “어디에 쓸 수 있을까?”로 변한다.
9. 보이는가·나눌 수 있는가·모을 수 있는가·행동할 수 있는가.
