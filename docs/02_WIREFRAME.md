# 3. 발표 모드

```text
┌─────────────────────────────────────────────────────────────────────┐
│ 판단하는 AI, 움직이는 수업       [장면 5/9] [00:13:20] [실습 모드] │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│                                                                     │
│              모델을 훈련시키는 일과                                │
│              행동을 설정하는 일은 다릅니다.                        │
│                                                                     │
│      [데이터 → 모델 훈련 → 판단]  ≠  [판단 결과 → 행동]             │
│                                                                     │
│                                                                     │
├─────────────────────────────────────────────────────────────────────┤
│ [◀ 이전]      [메모] [QR] [전체 화면] [타이머]            [다음 ▶] │
└─────────────────────────────────────────────────────────────────────┘
```

### 발표자 메모 패널

```text
권장 시간: 1분 30초

- 행동 연결 사이트 사례 소개
- “판단하는 뇌, 행동하는 몸통” 비유 사용
- 웹 출력에서 로봇 동작으로 확장
- 다음 장면: 학생들의 질문 변화
```

### 발표 모드 동작

- 키보드 좌우 방향키 이동
- Space 또는 PageDown 다음
- Escape 전체 화면 해제
- `P` 키로 발표자 메모 열기
- `Q` 키로 QR 코드 열기

# 13. 반응형 대응

## 1280px 이상

- 좌측 메뉴 + 중앙 콘텐츠 + 우측 패널
- 발표 모드 전체 화면 최적화

## 768~1279px

- 좌측 메뉴 축소
- 우측 패널 접기 가능
- 중앙 콘텐츠 폭 확대

## 767px 이하

- 단계 메뉴를 상단 드롭다운으로 변경
- 보조 패널을 본문 하단으로 이동
- Gem 빌더 입력과 출력을 세로 배치
- 하단 이전·다음 버튼 고정
- 교과 카드 한 열 배치
